# Digital Pet Design Specification

> Based on Codex hatch-pet skill, adapted for custom digital pet systems.

## 1. Visual Style

Codex digital pet 的统一美术风格：

- **像素风 / Chibi 比例** — 紧凑的 Q 版体型，夸张的头部比例
- **厚描边** — 深色 1-2px 轮廓线，可见的像素锯齿边缘
- **有限调色板** — 颜色数量克制，平涂赛璐璐上色
- **简洁表情** — 简单但富有表现力的五官
- **小肢体** — 微小的手臂/腿部，动作靠姿态变化而非细节

### 不要做的风格

- 精美插画 / 油画质感 / 动漫 Key Visual
- 3D 渲染 / 光泽 App Icon 风格
- 真实毛发 / 材质纹理 / 柔和渐变
- 高精度抗锯齿 / 复杂小配件

## 3. 动画状态定义

### 状态总览

| 行 | 状态 | 使用列 | 帧数 | 每帧时长 (ms) |
|----|------|--------|------|---------------|
| 0 | idle | 0-5 | 6 | 280, 110, 110, 140, 140, 320 |
| 1 | running-right | 0-7 | 8 | 120×7, 220 |
| 2 | running-left | 0-7 | 8 | 120×7, 220 |
| 3 | waving | 0-3 | 4 | 140×3, 280 |
| 4 | jumping | 0-4 | 5 | 140×4, 280 |
| 5 | failed | 0-7 | 8 | 140×7, 240 |
| 6 | waiting | 0-5 | 6 | 150×5, 260 |
| 7 | running | 0-5 | 6 | 120×5, 220 |
| 8 | review | 0-5 | 6 | 150×5, 280 |

### 各状态详细说明

#### Row 0 — idle（待机）
- **帧数**: 6
- **节奏**: 平静、低干扰的呼吸/眨眼循环
- **可接受动作**: 微妙呼吸、轻眨眼、轻微头部/身体晃动
- **禁止动作**: 挥手、行走、奔跑、跳跃、说话、工作、审查、大动作姿态变化、新道具

#### Row 1 — running-right（向右跑）
- **帧数**: 8
- **节奏**: 快速有方向感的移动循环
- **要求**: 通过身体、肢体、道具运动表现右移，不可绘制速度线、灰尘、地面阴影、运动轨迹

#### Row 2 — running-left（向左跑）
- **帧数**: 8
- **来源**: 可从 running-right 水平镜像得到（仅在宠物对称时），否则独立生成
- **要求**: 同 running-right，方向相反

#### Row 3 — waving（挥手）
- **帧数**: 4
- **节奏**: 清晰的挥手动作：举起、摆动、收回
- **要求**: 通过爪子姿态表现，禁止绘制波浪纹、运动弧线、闪光、符号

#### Row 4 — jumping（跳跃）
- **帧数**: 5
- **节奏**: 预备→起跳→顶峰→下落→着地
- **要求**: 通过身体位置表现垂直运动，禁止阴影、灰尘、着陆痕迹、冲击特效

#### Row 5 — failed（失败）
- **帧数**: 8
- **节奏**: 可读但不过于嘈杂的错误/沮丧反应
- **允许特效**: 眼泪（贴合面部）、小烟雾（贴合身体）、小星星（贴合身体）— 必须附着在宠物轮廓上
- **禁止特效**: 红色叉号、漂浮符号、分离的烟雾/星星/泪滴

#### Row 6 — waiting（等待）
- **帧数**: 6
- **节奏**: 耐心的待机变体
- **可接受动作**: 瞥视、小幅弹跳、道具轻微运动

#### Row 7 — running（运行中/忙碌）
- **帧数**: 6
- **含义**: 宠物正在忙于执行任务的工作状态循环
- **禁止**: 字面意义上的跑步/慢跑/冲刺/跑步机动作/高举膝盖/大步/摆臂/方向性移动

#### Row 8 — review（审查/思考）
- **帧数**: 6
- **节奏**: 专注/检查/思考的循环
- **可接受动作**: 倾身、眨眼、眼神聚焦、歪头、爪子位置变化
- **禁止**: 放大镜、纸张、代码、UI 面板、标点符号（除非该道具是宠物本身的一部分）

## 4. 特效规则

### 允许的特效（必须同时满足所有条件）

- 特效与动画状态相关，帮助表达动作含义
- 特效物理附着在宠物轮廓上（接触或重叠），不能漂浮在旁边
- 特效在同一格内，不产生独立精灵组件
- 特效为不透明、硬边缘、像素风格、非色度键颜色
- 特效足够小，在 192×208 内可读

**允许的例子**: 贴着脸的眼泪、贴着身体的烟雾、贴着身体的星星

### 禁止的特效

| 类别 | 具体项 |
|------|--------|
| 运动线 | 波浪纹、运动弧线、速度线、动作条纹、残影、模糊、拖影 |
| 漂浮物 | 分离的星星、闪光、漂浮标点、漂浮图标、落下的泪滴、分离的烟雾/灰尘 |
| 阴影/光效 | 投射阴影、接触阴影、地面阴影、着陆痕迹、冲击波、光晕、氛围光 |
| UI 元素 | 文字、标签、帧号、网格线、对话框、UI 面板、代码片段 |
| 背景 | 棋盘格透明、白色背景、黑色背景、风景 |
| 色度问题 | 色度键相近颜色出现在宠物/道具/特效/高光/阴影中 |
| 像素问题 | 散落像素、断开轮廓、裁切身体、姿势重叠、越界到相邻格子 |

## 5. 透明背景与色度键

### 生成流程

1. 在纯色色度键背景上生成宠物图像
2. 默认色度键: `#00ff00`（绿色宠物用 `#ff00ff`）
3. 生成后用 `remove_chroma_key.py` 移除背景，输出带 Alpha 通道的 PNG/WebP

### 色度键要求

- 背景必须为均匀单色，无阴影、渐变、纹理、反光
- 宠物主体完全分离于背景，边缘锐利，留有充裕边距
- 宠物任何部位不得使用色度键颜色
- 无投射阴影、接触阴影、倒影、水印

## 7. QA 验收标准

### 几何校验

- [ ] 精确 1536×1872 尺寸
- [ ] 8 列 × 9 行网格
- [ ] 每帧在 192×208 格子内
- [ ] 未使用格子完全透明
- [ ] 无标签、间距、边框、网格线、格外语素

### 角色一致性

- [ ] 所有行动画中保持相同轮廓和比例
- [ ] 相同面部和表情语言
- [ ] 相同材质、调色板、光照、道具设计
- [ ] 无任何帧引入新的非预期角色或物体

### 精灵图风格

- [ ] 美术风格读作 Codex 数字宠物精灵，非精美插画或光泽 App Icon
- [ ] 轮廓紧凑、块状，在 192×208 格内可读
- [ ] 轮廓线深色简单，可见像素锯齿边缘
- [ ] 调色板有限，平涂赛璐璐上色
- [ ] 无油画质感、真实材质细节、柔和渐变、高精度抗锯齿

### 动画完整性

- [ ] 每行使用精确的预期帧数
- [ ] 首尾帧可无缝循环（无明显跳变）
- [ ] 方向性行读作预期方向
- [ ] 各状态动作在宠物尺寸下可识别
- [ ] 帧是不同的动画变体，非同一源图重复

### 应用适配

- [ ] idle 第一帧可作为静态 reduced-motion 宠物
- [ ] idle 行平静低干扰，不会被误读为其他状态
- [ ] 无重要细节过小不可读
- [ ] 无帧被格子裁切
- [ ] failed/review/waiting 状态与普通 idle 有明显区分
- [ ] 无色度键残留（角色区域被误抠则需更换色度键重生成）
- [ ] 无边缘碎片或相邻精灵渗透
- [ ] 无色度键颜色作为阴影/灰尘/拖影/光晕出现

### 修复策略

按最小范围优先修复：

1. **单帧修复** — 仅某一帧有问题
2. **单行修复** — 某一行动画整体有问题
3. **全图重新生成** — 仅当角色身份或布局广泛性错误时

## 8. 生成工作流

### 工具链

| 步骤 | 工具 | 说明 |
|------|------|------|
| 基础形象生成 | imagegen / AI 图像生成 | 生成宠物主视觉，作为所有帧的视觉锚点 |
| 各状态帧生成 | imagegen / AI 图像生成 | 以基础形象为参考，逐状态生成行条带 |
| 背景移除 | remove_chroma_key.py | 色度键抠图，输出透明 PNG |
| 精灵图拼合 | finalize_pet_run.py | 按网格规格组装 atlas |
| 校验 | validate_atlas.py | 几何、透明度、帧数检查 |
| QA 预览 | contact-sheet + video | 缩略图总览 + 逐状态动画预览 |
| 打包 | package_custom_pet.py | 输出最终 pet.json + spritesheet.webp |

### 生成数量估算

| 类型 | 数量 |
|------|------|
| 基础形象 | 1 张 |
| 行条带（9 个状态） | 9 张 |
| **总计生成调用** | **~10 次** |
